# Go-Live Checklist

## Tujuan

Checklist singkat ini dipakai saat hari eksekusi go-live backend monorepo ke cPanel.

Gunakan bersama:
- [AUDIT KESIAPAN MONOREPO.md](/e:/thechoosentalksnext/docs/core/architecture/laravel-hybrid/AUDIT%20KESIAPAN%20MONOREPO.md)
- [FIRST DEPLOY RUNBOOK.md](/e:/thechoosentalksnext/docs/core/architecture/laravel-hybrid/FIRST%20DEPLOY%20RUNBOOK.md)

## A. Sebelum Menekan Workflow

- [ ] SSH ke server cPanel berhasil
- [ ] user SSH adalah `thechoosentalks`
- [ ] deploy path benar:
  - `/home/thechoosentalks/deploy/apps/thechoosentalks`
- [ ] `shared/.env` ada
- [ ] `DB_CONNECTION=mysql`
- [ ] `APP_ENV=production`
- [ ] `APP_DEBUG=false`
- [ ] `pdo_mysql` aktif
- [ ] `tar`, `rsync`, `sha256sum` tersedia
- [ ] `public_html` writable
- [ ] public key GitHub Actions sudah ada di `authorized_keys`
- [ ] secret GitHub sudah lengkap:
  - `CPANEL_SSH_HOST`
  - `CPANEL_SSH_PORT`
  - `CPANEL_SSH_USER`
  - `CPANEL_DEPLOY_PATH`
  - `CPANEL_SSH_KEY`

## B. Command Preflight Cepat di Server

Jalankan:

```bash
whoami
hostname
mkdir -p /home/thechoosentalks/deploy/apps/thechoosentalks/releases
mkdir -p /home/thechoosentalks/deploy/apps/thechoosentalks/shared
test -f /home/thechoosentalks/deploy/apps/thechoosentalks/shared/.env && echo "shared .env OK" || echo "shared .env MISSING"
grep -E '^(APP_ENV|APP_DEBUG|APP_URL|DB_CONNECTION|DB_HOST|DB_PORT|DB_DATABASE|DB_USERNAME|CORS_ALLOWED_ORIGINS|FIREBASE_PROJECT_ID)=' /home/thechoosentalks/deploy/apps/thechoosentalks/shared/.env
php -m | grep pdo_mysql
command -v tar
command -v rsync
command -v sha256sum
touch ~/public_html/preflight-write-test.txt
rm -f ~/public_html/preflight-write-test.txt
test -f ~/.ssh/authorized_keys && echo "authorized_keys OK" || echo "authorized_keys MISSING"
```

Lanjut hanya jika semua hasil sehat.

## C. Jalankan Workflow Backend Pertama

- [ ] buka GitHub repo `TCT--Hybrid--Monorepo`
- [ ] buka tab `Actions`
- [ ] pilih `Backend cPanel Deploy`
- [ ] pilih branch `main`
- [ ] klik `Run workflow`

## D. Step GitHub Actions yang Harus Hijau

- [ ] `Checkout repository`
- [ ] `Setup Node.js`
- [ ] `Setup PHP`
- [ ] `Validate required deploy secrets`
- [ ] `Install backend PHP dependencies`
- [ ] `Install backend frontend dependencies`
- [ ] `Build backend assets`
- [ ] `Create backend release artifact`
- [ ] `Start SSH agent`
- [ ] `Upload artifact and deploy scripts`
- [ ] `Run remote deploy`

## E. Verifikasi Setelah Workflow Hijau

Login ke server lalu jalankan:

```bash
ls -la /home/thechoosentalks/deploy/apps/thechoosentalks
readlink -f /home/thechoosentalks/deploy/apps/thechoosentalks/current
cd /home/thechoosentalks/deploy/apps/thechoosentalks/current
php artisan about
php artisan tinker --execute="echo \App\Models\User::count();"
ls -la ~/public_html
curl -I https://thechoosentalks.org/
curl -I https://thechoosentalks.org/up
```

## F. Tanda Sukses

- [ ] `current` pindah ke release timestamp baru
- [ ] `php artisan about` normal
- [ ] engine MariaDB hidup
- [ ] website utama tetap `200`
- [ ] healthcheck tidak rollback

## G. Tanda Gagal

- [ ] workflow merah di GitHub Actions
- [ ] `current` tidak pindah
- [ ] `current` pindah lalu rollback
- [ ] `php artisan about` gagal
- [ ] HTTP healthcheck gagal

## H. Jika Gagal

1. baca step merah di GitHub Actions
2. login ke server
3. cek:

```bash
ls -la /home/thechoosentalks/deploy/apps/thechoosentalks
readlink -f /home/thechoosentalks/deploy/apps/thechoosentalks/current
```

4. jika perlu rollback manual:

```bash
bash /home/thechoosentalks/deploy/apps/thechoosentalks/rollback.sh
```

- [ ] uji endpoint:

  - `/api/v1/community/posts`
- [ ] uji auth sync Firebase jika sudah diaktifkan
- [ ] lanjut ke release frontend melalui `frontend-prod`
