# First Deploy Runbook

## Tujuan

Dokumen ini adalah urutan operasional setelah preflight server lolos, sampai workflow backend pertama dijalankan dari GitHub Actions dan hasilnya diverifikasi.

Repo target:
- `https://github.com/engelwillem/TCT--Hybrid--Monorepo.git`

Workflow target:
- [backend-cpanel-deploy.yml](/e:/thechoosentalksnext/.github/workflows/backend-cpanel-deploy.yml)

## Prasyarat

Jangan mulai jika preflight belum lolos penuh di:
- [AUDIT KESIAPAN MONOREPO.md](/e:/thechoosentalksnext/docs/core/architecture/laravel-hybrid/AUDIT%20KESIAPAN%20MONOREPO.md)

Minimal yang harus sudah benar:
- SSH login ke server berhasil
- `authorized_keys` sudah siap
- deploy path writable
- `shared/.env` ada
- `DB_CONNECTION=mysql`
- `pdo_mysql` aktif
- `public_html` writable

## 1. Finalisasi Secret di GitHub

Masuk ke:
- `GitHub repo`
- `Settings`
- `Secrets and variables`
- `Actions`

Buat secret ini:

```text
CPANEL_SSH_HOST=209.42.27.90
CPANEL_SSH_PORT=22
CPANEL_SSH_USER=thechoosentalks
CPANEL_DEPLOY_PATH=/home/thechoosentalks/deploy/apps/thechoosentalks
CPANEL_SSH_KEY=<private key OpenSSH>
```

Verifikasi:
- `CPANEL_SSH_KEY` adalah private key
- bukan `.pub`
- `CPANEL_DEPLOY_PATH` sama persis dengan path server

## 2. Pastikan Workflow Sudah Ada di Branch `main`

File yang harus ada:
- [backend-cpanel-deploy.yml](/e:/thechoosentalksnext/.github/workflows/backend-cpanel-deploy.yml)
- [frontend-monorepo-checks.yml](/e:/thechoosentalksnext/.github/workflows/frontend-monorepo-checks.yml)

Pastikan perubahan sudah di-push ke `main`.

## 3. Pastikan Server Siap Menerima Artifact

Sebelum run pertama, login SSH dan jalankan:

```bash
mkdir -p /home/thechoosentalks/deploy/apps/thechoosentalks/releases
mkdir -p /home/thechoosentalks/deploy/apps/thechoosentalks/shared
ls -la /home/thechoosentalks/deploy/apps/thechoosentalks
test -f /home/thechoosentalks/deploy/apps/thechoosentalks/shared/.env && echo "shared .env OK"
```

## 4. Jalankan Workflow Backend Pertama

Masuk ke:
- `GitHub repo`
- `Actions`
- pilih `Backend cPanel Deploy`
- klik `Run workflow`
- pilih branch: `main`
- jalankan

Jangan gabungkan dengan deploy frontend pertama. Fokus backend saja dulu.

## 5. Urutan Hasil yang Harus Muncul di GitHub Actions

Pada run yang sehat, Anda akan melihat langkah ini lolos:

1. `Checkout repository`
2. `Setup Node.js`
3. `Setup PHP`
4. `Validate required deploy secrets`
5. `Install backend PHP dependencies`
6. `Install backend frontend dependencies`
7. `Build backend assets`
8. `Create backend release artifact`
9. `Start SSH agent`
10. `Upload artifact and deploy scripts`
11. `Run remote deploy`

Jika semua hijau, lanjut ke verifikasi server.

## 6. Cara Membaca Kegagalan

### Gagal di `Validate required deploy secrets`

Makna:
- ada secret GitHub yang kosong atau salah nama

Tindakan:
- cek ulang nama secret
- cek tidak ada typo

### Gagal di `Install backend PHP dependencies`

Makna:
- `composer.json` valid tetapi dependency gagal di CI
- bisa karena ext PHP kurang atau dependency conflict

Tindakan:
- baca log Composer
- pastikan `setup-php` sudah memberi `pdo_mysql`

### Gagal di `Install backend frontend dependencies`

Makna:
- `backend-api/package-lock.json` atau `package.json` belum sinkron

Tindakan:
- audit lockfile backend

### Gagal di `Build backend assets`

Makna:
- Vite/Laravel frontend backend gagal build

Tindakan:
- baca error compile
- pastikan `public/build/manifest.json` memang bisa dihasilkan

### Gagal di `Upload artifact and deploy scripts`

Makna:
- SSH host/port/user/key/path salah
- atau server menolak koneksi

Tindakan:
- cek `CPANEL_SSH_HOST`
- cek `CPANEL_SSH_USER`
- cek key di `authorized_keys`
- cek path deploy

### Gagal di `Run remote deploy`

Makna:
- koneksi SSH sukses, tetapi `deploy.sh` gagal di server

Tindakan:
- baca log step ini paling detail
- biasanya masalah ada di:
  - `shared/.env`
  - koneksi MariaDB
  - permission `public_html`
  - `vendor/autoload.php` hilang
  - `public/build/manifest.json` hilang

## 7. Verifikasi Server Setelah Run Sukses

Login SSH ke server lalu jalankan:

```bash
ls -la /home/thechoosentalks/deploy/apps/thechoosentalks
readlink -f /home/thechoosentalks/deploy/apps/thechoosentalks/current
ls -la /home/thechoosentalks/deploy/apps/thechoosentalks/releases
```

Hasil yang diharapkan:
- ada folder release timestamp baru
- `current` menunjuk ke release terbaru

## 8. Verifikasi Laravel di Release Aktif

Jalankan:

```bash
cd /home/thechoosentalks/deploy/apps/thechoosentalks/current
php artisan about
php artisan tinker --execute="echo \App\Models\User::count();"
```

Hasil yang diharapkan:
- Laravel bootstrap normal
- koneksi MariaDB hidup
- jumlah user tidak nol secara tidak wajar

## 9. Verifikasi Web Root

Jalankan:

```bash
ls -la ~/public_html
php -r "echo file_exists(getenv('HOME').'/public_html/index.php') ? 'index.php OK' : 'index.php MISSING';"
```

Hasil yang diharapkan:
- `index.php` ada
- asset publik backend sudah tersalin

## 10. Verifikasi HTTP Dasar

Jalankan dari server atau browser:

```bash
curl -I https://thechoosentalks.org/
curl -I https://thechoosentalks.org/up
```

Jika API publik memang diarahkan ke origin ini, uji juga:

```bash
curl -I https://thechoosentalks.org/api/v1/today
curl -I https://thechoosentalks.org/api/v1/community/posts
```

## 11. Tanda Deploy Pertama Sukses

Deploy pertama dianggap sukses jika:
- workflow GitHub hijau penuh
- `current` mengarah ke release baru
- `php artisan about` normal
- koneksi MariaDB hidup
- website utama tetap merespons
- healthcheck tidak memicu rollback

## 12. Tanda Deploy Pertama Gagal

Deploy pertama dianggap gagal jika:
- workflow berhenti sebelum upload
- workflow berhenti di remote deploy
- `current` tidak berpindah
- `current` berpindah lalu rollback ke release lama
- `php artisan about` gagal
- HTTP healthcheck gagal

## 13. Jika Gagal, Langkah Pertama yang Dilakukan

1. buka log GitHub Actions step yang merah
2. identifikasi apakah gagal di:
   - secret
   - SSH
   - build
   - deploy script
   - database
3. login ke server
4. cek:

```bash
ls -la /home/thechoosentalks/deploy/apps/thechoosentalks
readlink -f /home/thechoosentalks/deploy/apps/thechoosentalks/current
```

5. jika perlu, jalankan inspeksi release aktif:

```bash
cd /home/thechoosentalks/deploy/apps/thechoosentalks/current
php artisan about
```

## 14. Jika Perlu Rollback Manual

Jalankan di server:

```bash
bash /home/thechoosentalks/deploy/apps/thechoosentalks/rollback.sh
```

Atau target release tertentu:

```bash
bash /home/thechoosentalks/deploy/apps/thechoosentalks/rollback.sh --release /home/thechoosentalks/deploy/apps/thechoosentalks/releases/<timestamp>
```

Baru lanjut ke:
- uji auth sync Firebase -> Sanctum
- uji `GET/POST /api/v1/community/posts`
- uji frontend terhadap backend produksi
- finalisasi branch release frontend `frontend-prod`
