**Migration Order**

1. VerseHub API surface.
2. Inbox/DM API.
3. Profile/settings/auth lifecycle API.
4. Study Paths API.
5. Channels/Sabbath School API.
6. Remove mock fallback from Next pages.
7. Freeze Laravel `web.php` public app routes as legacy only.

**Risk Notes**

- `next build` currently passes, but build validation is weakened because [next.config.ts](/e:/thechoosentalksnext/next.config.ts) ignores TS and ESLint during build.
- If you keep mock fallbacks in production, integration bugs will look like “works visually” while data parity is actually broken.
- `src/app/layout.tsx` still assumes app-wide Firebase bootstrap, so auth ownership needs to stay explicit between Firebase client auth and Laravel Sanctum token issuance in [src/app/layout.tsx](/e:/thechoosentalksnext/src/app/layout.tsx).

Kalau Anda mau, saya bisa lanjutkan jadi **dokumen implementasi per endpoint**: mapping `route lama Laravel -> API JSON baru -> halaman Next yang mengonsumsi -> payload request/response`.
