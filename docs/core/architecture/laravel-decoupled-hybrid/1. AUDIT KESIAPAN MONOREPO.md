# Audit Kesiapan Monorepo

## Preflight Server untuk cPanel SSH

Dokumen ini adalah urutan eksekusi terminal yang praktis untuk preflight sebelum workflow backend pertama kali dijalankan dari GitHub Actions.

Gunakan di terminal SSH cPanel sebagai user:
- `thechoosentalks`

Target path:
- `/home/thechoosentalks/deploy/apps/thechoosentalks`

## 1. Verifikasi Identitas Server

Jalankan:

```bash
whoami
hostname
pwd
```

Hasil yang diharapkan:
- user adalah `thechoosentalks`
- Anda benar-benar masuk ke server target

## 2. Verifikasi Folder Deploy

Jalankan:

```bash
mkdir -p /home/thechoosentalks/deploy/apps/thechoosentalks/releases
mkdir -p /home/thechoosentalks/deploy/apps/thechoosentalks/shared
ls -la /home/thechoosentalks/deploy/apps/thechoosentalks
```

Hasil yang diharapkan:
- folder `releases` ada
- folder `shared` ada
- path deploy bisa ditulis oleh user SSH

## 3. Verifikasi Shared `.env`

Jalankan:

```bash
test -f /home/thechoosentalks/deploy/apps/thechoosentalks/shared/.env && echo "shared .env OK" || echo "shared .env MISSING"
grep -E '^(APP_ENV|APP_DEBUG|APP_URL|DB_CONNECTION|DB_HOST|DB_PORT|DB_DATABASE|DB_USERNAME|CORS_ALLOWED_ORIGINS|FIREBASE_PROJECT_ID)=' /home/thechoosentalks/deploy/apps/thechoosentalks/shared/.env
```

Hasil yang diharapkan:
- `shared .env OK`
- `DB_CONNECTION=mysql`
- `APP_ENV=production`
- `APP_DEBUG=false`

Jangan cetak `DB_PASSWORD` ke terminal bersama.

## 4. Verifikasi PHP dan Tooling Server

Jalankan:

```bash
php -v
php -m | grep pdo_mysql
command -v tar
command -v rsync
command -v sha256sum
```

Hasil yang diharapkan:
- PHP tersedia
- `pdo_mysql` aktif
- `tar`, `rsync`, dan `sha256sum` tersedia

## 5. Verifikasi Write Access ke `public_html`

Jalankan:

```bash
touch ~/public_html/preflight-write-test.txt
ls -l ~/public_html/preflight-write-test.txt
rm -f ~/public_html/preflight-write-test.txt
```

Hasil yang diharapkan:
- file test berhasil dibuat dan dihapus

## 6. Verifikasi SSH Key Sudah Siap

Jalankan:

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
ls -la ~/.ssh
test -f ~/.ssh/authorized_keys && echo "authorized_keys OK" || echo "authorized_keys MISSING"
chmod 600 ~/.ssh/authorized_keys 2>/dev/null || true
```

Hasil yang diharapkan:
- `authorized_keys OK`

Catatan:
- GitHub Actions akan memakai private key
- server hanya menyimpan public key di `authorized_keys`

## 7. Verifikasi Script Deploy Sudah Bisa Ditempatkan

Jalankan:

```bash
touch /home/thechoosentalks/deploy/apps/thechoosentalks/deploy.sh
touch /home/thechoosentalks/deploy/apps/thechoosentalks/healthcheck.sh
touch /home/thechoosentalks/deploy/apps/thechoosentalks/rollback.sh
chmod 700 /home/thechoosentalks/deploy/apps/thechoosentalks/deploy.sh
chmod 700 /home/thechoosentalks/deploy/apps/thechoosentalks/healthcheck.sh
chmod 700 /home/thechoosentalks/deploy/apps/thechoosentalks/rollback.sh
ls -l /home/thechoosentalks/deploy/apps/thechoosentalks
```

Hasil yang diharapkan:
- file script bisa dibuat
- permission script bisa diset

Catatan:
- kalau file asli belum di-upload, ini hanya uji permission path

## 8. Verifikasi MySQL Siap Dipakai

Jalankan:

```bash
php -m | grep -E 'pdo_mysql|mysqli'
```

Hasil yang diharapkan:
- minimal `pdo_mysql` muncul

Jika release aktif backend sudah ada, jalankan juga:

```bash
cd /home/thechoosentalks/deploy/apps/thechoosentalks/current
php artisan about
php artisan tinker --execute="echo \App\Models\User::count();"
```

Hasil yang diharapkan:
- Laravel bisa bootstrap
- koneksi database hidup

## 9. Verifikasi `public_html/index.php` Bisa Diganti

Jalankan:

```bash
cp ~/public_html/index.php ~/public_html/index.php.preflight.bak 2>/dev/null || true
printf '%s\n' '<?php echo "preflight";' > ~/public_html/index.php.preflight.test
ls -l ~/public_html/index.php.preflight.test
rm -f ~/public_html/index.php.preflight.test
```

Hasil yang diharapkan:
- file test bisa ditulis di `public_html`

## 10. Verifikasi Nilai GitHub Secret yang Akan Dipakai

Cocokkan secara manual:

```text
CPANEL_SSH_HOST=209.42.27.90
CPANEL_SSH_PORT=22
CPANEL_SSH_USER=thechoosentalks
CPANEL_DEPLOY_PATH=/home/thechoosentalks/deploy/apps/thechoosentalks
CPANEL_SSH_KEY=<private key OpenSSH>
```

Checklist:
- `CPANEL_SSH_HOST` bukan domain web yang salah
- `CPANEL_SSH_KEY` adalah private key, bukan `.pub`
- deploy path sama persis dengan path server yang tadi diuji

## 11. Preflight Ringkas Satu Kali Jalan

Jika ingin cek cepat dalam satu blok:

```bash
echo "== identity =="
whoami
hostname
echo "== deploy path =="
mkdir -p /home/thechoosentalks/deploy/apps/thechoosentalks/releases
mkdir -p /home/thechoosentalks/deploy/apps/thechoosentalks/shared
ls -ld /home/thechoosentalks/deploy/apps/thechoosentalks /home/thechoosentalks/deploy/apps/thechoosentalks/releases /home/thechoosentalks/deploy/apps/thechoosentalks/shared
echo "== shared env =="
test -f /home/thechoosentalks/deploy/apps/thechoosentalks/shared/.env && echo "shared .env OK" || echo "shared .env MISSING"
grep -E '^(APP_ENV|APP_DEBUG|APP_URL|DB_CONNECTION|DB_HOST|DB_PORT|DB_DATABASE|DB_USERNAME|CORS_ALLOWED_ORIGINS|FIREBASE_PROJECT_ID)=' /home/thechoosentalks/deploy/apps/thechoosentalks/shared/.env
echo "== php tooling =="
php -v
php -m | grep pdo_mysql
command -v tar
command -v rsync
command -v sha256sum
echo "== public_html write =="
touch ~/public_html/preflight-write-test.txt
ls -l ~/public_html/preflight-write-test.txt
rm -f ~/public_html/preflight-write-test.txt
echo "== ssh auth =="
test -f ~/.ssh/authorized_keys && echo "authorized_keys OK" || echo "authorized_keys MISSING"
```

## 12. Kriteria Aman untuk Menekan Workflow Pertama

Lanjut jalankan workflow hanya jika:
- `shared .env OK`
- `DB_CONNECTION=mysql`
- `pdo_mysql` aktif
- deploy path writable
- `public_html` writable
- `authorized_keys OK`

Jangan jalankan workflow jika:
- SSH host belum pasti
- `shared/.env` belum ada
- database masih salah atau belum siap
- user SSH tidak bisa menulis ke deploy path atau `public_html`
