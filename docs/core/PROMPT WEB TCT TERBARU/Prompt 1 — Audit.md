# PROMPT

Prompt 1 — Audit + Reframing /today

Anda adalah senior product designer, UX writer, dan iOS product strategist.

Saya sedang membangun website rohani premium bernama The Chosen Talks. Saya ingin halaman /today menjadi pusat pengalaman utama produk ini.

Konteks produk:
- Target: orang Kristen / user yang ingin kembali setiap hari
- Goal utama: high retention
- Prioritas pengalaman: refleksi (utama), doa (kedua), pertumbuhan (jangka panjang)
- Saya tidak ingin halaman terasa seperti blog, portal, atau landing page
- Saya ingin pengalaman terasa seperti iOS native app premium: tenang, mahal, elegan, minim noise, sangat fokus
- Saya ingin user merasa “didampingi”, bukan “diceramahi”
- Tone harus calm, premium, intimate, spiritually grounded

Tolong audit dan reframe halaman /today sebagai daily spiritual companion.

Tugas Anda:
1. Jelaskan apa seharusnya fungsi inti /today dalam produk ini
2. Jelaskan kesalahan umum yang membuat halaman seperti ini terasa noisy dan tidak retentive
3. Tentukan prinsip desain utama untuk /today
4. Tentukan “one primary user action” yang harus dilakukan user setiap kali membuka /today
5. Berikan arah produk yang sangat tegas: apa yang harus jadi fokus, apa yang harus dikurangi, apa yang harus dihilangkan

Aturan:
- Jangan beri jawaban generik
- Jangan terlalu teknis
- Fokus pada product thinking, UX flow, dan emotional experience
- Jawaban harus konkret dan decision-oriented
- Formatkan dengan section yang jelas:
  1. Core Role of /today
  2. Key Problems to Avoid
  3. Design Principles
  4. Primary User Action
  5. Strategic Direction
Prompt 2 — Exact Layout /today
Sekarang lanjutkan. Jangan ulangi penjelasan sebelumnya.

Berdasarkan konteks yang sudah saya berikan, buatkan exact layout untuk halaman /today.

Tujuan halaman:
- menjadi ritual harian
- terasa seperti iOS native premium app
- minim noise
- tidak terasa seperti website penuh teks
- user langsung tahu harus melakukan apa
- flow utama: receive → reflect → pray → complete

Yang saya butuhkan:
1. Struktur exact halaman dari atas ke bawah
2. Urutan section yang paling ideal
3. Fungsi setiap section
4. Mana yang wajib tampil above the fold
5. Mana yang harus disembunyikan / dipindah / dijadikan secondary
6. Rekomendasi panjang teks maksimal per section
7. Rekomendasi hierarchy visual per section

Batasan:
- Hanya 1 primary CTA
- Tidak boleh ada terlalu banyak pilihan
- Tidak boleh terasa seperti feed
- Tidak boleh terasa seperti artikel panjang
- Desain harus cocok untuk mobile-first
- Harus terasa seperti layar utama iPhone app premium

Format output:
1. Page Goal
2. Layout Overview
3. Section-by-Section Breakdown
4. Above-the-Fold Structure
5. What to Remove
6. Why This Layout Supports Retention

Tolong sangat konkret, bukan abstrak.
