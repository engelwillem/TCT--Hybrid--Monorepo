# Docs Structure

Dokumentasi dibagi menjadi empat area utama:

- `core/`: dokumen aktif proyek yang perlu masuk git.
- `reference/`: snapshot konfigurasi dan referensi teknis yang masih relevan.
- `archive/`: arsip historis, bundle, audit visual, dan referensi lama.
- `quarantine/`: dump sensitif, state editor, dan artefak sementara yang tidak boleh masuk git.

Aturan praktis:

- Gunakan `docs/core/` untuk blueprint, implementation plan, parity, dan walkthrough aktif.
- Gunakan `docs/reference/` untuk snapshot konfigurasi yang kadang perlu dirujuk.
- Jangan taruh dokumen kerja aktif baru di `docs/archive/` atau `docs/quarantine/`.
