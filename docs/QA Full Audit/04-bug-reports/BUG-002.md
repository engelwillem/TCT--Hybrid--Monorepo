# Bug Report: BUG-002

## Title
Broken Login API Endpoint Call (POST /api/auth/login -> 404).

## Severity
🛑 **Blocker**

## Priority
Immediate

## Suggested Owner: Codex

## Area / Module
Auth / API Integration

## Environment
Production (`https://www.thechoosentalks.org/login`)

## Preconditions
User attempts to login with valid or invalid credentials.

## Steps to Reproduce
1. Go to `/login`.
2. Fill Email and Password.
3. Click "Buka Blokir" (Login button).
4. Inspect Network/Console.

## Expected Result
Request should be sent to a valid Laravel API endpoint (e.g., `/api/v1/login`) and return auth tokens or validation errors.

## Actual Result
Request is sent to `https://www.thechoosentalks.org/api/auth/login` and server returns **404 Not Found**.

## Impact
Zero users can authenticate. Application is non-functional for registered users.

## Evidence
[Console Error: 404 @ /api/auth/login]

## Recommendation
Update the API endpoint URL in the auth service to match the backend production contract.

## Validation Checklist
- [ ] Login returns 200/401, not 404.
- [ ] User is redirected to `/today` or intended destination after success.
