# Bug Report: BUG-003

## Title
Bottom Nav conflicts with Verse Sheet in VerseHub Reader.

## Severity
🔴 **High** (UX/Product Blocker per Owner request)

## Priority
High

## Area / Module
VerseHub / UI Shell

## Environment
Production (`https://www.thechoosentalks.org/versehub/id?ref=mzm-23`)

## Preconditions
User navigates to a chapter in VerseHub.

## Steps to Reproduce
1. Go to `https://www.thechoosentalks.org/versehub/id`.
2. Select any chapter (e.g. Mazmur 23).
3. Click on any verse text (e.g. Verse 1).
4. Observe the bottom of the screen.

## Expected Result
Bottom Navigation should be hidden/suppressed to give full focus to the Verse Action Sheet.

## Actual Result
Bottom Navigation remains visible and overlays or sits behind the Verse Action Sheet, creating visual clutter and conflicting touch targets.

## Impact
Reduces clarity and breaks the design requirement for "quiet, focused reading".

## Evidence
[VerseHub Snapshot: L197-210]

## Recommendation
Implement a `hideNavigation` flag or a CSS class to suppress the `AppShell` navigation when `isVerseSheetOpen` is true.
