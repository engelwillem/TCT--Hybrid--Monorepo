# Bug Report: BUG-005

## Title
Community "Bagikan" button is silently disabled for Guests without UX feedback.

## Severity
🟠 **Medium** (UX Improvement)

## Priority
Medium

## Area / Module
Community / Post Editor

## Environment
Production (`https://www.thechoosentalks.org/community`)

## Preconditions
User is a Guest (not logged in).

## Steps to Reproduce
1. Go to `https://www.thechoosentalks.org/community`.
2. Click "Mulai menulis...".
3. Type a post body.
4. Observe the "Bagikan" button state.

## Expected Result
1. The button should ideally be active, and trigger a Login Modal/Redirect on click.
2. OR if disabled, a tooltip/message should plainly state "Please login to share your reflection."

## Actual Result
The "Bagikan" button remains disabled (`[disabled]`). No explanation or call-to-action is provided to the Guest user.

## Impact
Poor user experience; guests feel blocked but don't know the required action.

## Evidence
[Community Snapshot 12029: L81-156]

## Recommendation
Implement a conditional `showAuthGate` on button click or replace the disabled button with a "Login to Share" button for Guests.
