# Future UX Direction

## What This Website Should Become
Target paling tepat untuk website ini adalah:

`a focused spiritual companion`

Artinya, produk ini bukan diarahkan menjadi:
- blog konten
- social network umum
- katalog ayat
- dashboard modul yang serba ada

Melainkan menjadi tempat harian untuk:
- grounding
- membaca
- merenung
- bertumbuh
- berbagi

## Product Promise
Saat user membuka website ini, mereka harus merasa:

`Saya tidak datang untuk menjelajah kekacauan fitur. Saya datang untuk dibantu mengambil langkah rohani berikutnya.`

## Core Identity

### 1. Calm
UI harus terasa tenang, lapang, dan tidak agresif.

### 2. Guided
Produk tidak pasif. Ia harus membantu user memilih langkah berikutnya.

### 3. Scripture-centered
VerseHub dan layer refleksi harus tetap menjadi pusat kedalaman produk.

### 4. Social but not noisy
Community harus terasa suportif dan penuh konteks, bukan ramai tanpa arah.

### 5. Habit-forming
Paths dan Today harus membentuk ritme, bukan kunjungan sesekali.

## Final Experience Model

### Layer 1 — Public Layer
Landing menjelaskan:
- mengapa produk ini ada
- siapa yang cocok memakainya
- bagaimana alurnya dari hari ke hari

### Layer 2 — Daily Layer
Today menjadi:
- layar pembuka utama
- penentu prioritas
- pemilih langkah tercepat

### Layer 3 — Deep Layer
Permukaan terdalam:
- VerseHub: membaca dan memahami
- Community: merespons dan membagikan
- Paths: melanjutkan pertumbuhan
- Profile: mengatur konteks personal

## Canonical User Loop
Loop utama produk nantinya harus terasa seperti ini:

1. User masuk ke `Today`
2. User menerima konteks dan prioritas
3. User pindah ke `VerseHub` atau `Paths`
4. User menulis respons di `Community`
5. User kembali lagi besok ke `Today`

## Canonical Navigation Model

### Primary
- Today
- VerseHub
- Paths
- Community
- Profile

### Secondary
- Inbox
- Composer
- Reader subviews
- auth pages

### Transitional / To Be Retired
- Channels
- Reflections standalone
- Library
- Visitors
- Gate updates

## Visual Direction

### V1 Direction
- Dawn / light-first
- large radii
- soft premium surfaces
- semantic spacing
- gentle motion

### What To Avoid
- dashboard density
- noisy marketing gradients inside app surfaces
- duplicated visual language between old routes and new shells
- too many equal CTAs in one screen

## UX Principles Going Forward

### 1. Every page must have one primary job
Setiap layar harus punya tujuan dominan yang jelas.

### 2. Every page must know the next handoff
Tidak boleh ada surface yang terasa dead-end.

### 3. The app should feel emotionally aware
Today dan response layers harus terasa membaca keadaan user, bukan hanya menampilkan data.

### 4. The Bible experience should feel premium
VerseHub harus menjadi salah satu permukaan paling polished di seluruh produk.

### 5. Reduction is part of the strategy
Menghapus atau menggabungkan route bukan kehilangan fitur. Itu justru bagian dari memperjelas produk.

## Near-Term Direction

### Now
- perkuat Today sebagai decision surface
- haluskan VerseHub sebagai reader-first experience
- pertegas Community sebagai response layer

### Next
- satukan bahasa visual landing dan area aplikasi
- kurangi drift istilah dan modul
- de-emphasize route yang tidak lagi inti

### Later
- capai loop penuh harian: relevance -> reading -> reflection -> community -> progress

## Final Statement
Website ini nantinya seharusnya menjadi:

`the daily spiritual home screen for its users`

Bukan tempat untuk melihat banyak fitur, tetapi tempat untuk mengambil satu langkah yang paling penting hari ini.
